# Security Policy

## Supported Versions

| Version | Supported |
| --- | --- |
| 0.1.x | Yes |

## Reporting a Vulnerability

- Do not open a public issue for a suspected vulnerability.
- Report security concerns to `csmathguy@gmail.com`.
- If the issue relates to sensitive data handling, describe the workflow and impact without sending raw tax documents, extracted fields, or generated filings.

## Repository-Specific Expectations

- This project is intended for local-only tax workflows.
- Test fixtures must be synthetic or fully redacted.
- Secrets, private keys, tax records, and generated returns must never be committed.
