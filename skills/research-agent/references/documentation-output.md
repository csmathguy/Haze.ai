# Documentation Output

## Structure The Result

When research turns into repository documentation:

1. Choose whether the page is a tutorial, how-to, reference, or explanation.
2. Put the audience, scope, and time-sensitive qualifiers near the top.
3. Lead with the answer or key takeaway before supporting detail.

## Writing Rules

- Make headings specific and scannable.
- Keep paragraphs to one topic.
- Use numbered steps for procedures.
- Use bullets for unordered constraints or checklists.
- Keep list items parallel.
- Prefer short, direct sentences and concrete nouns.

## Citation Discipline

- Preserve exact dates, versions, and tax years.
- Identify when a point is sourced fact versus inference.
- Update `docs/research-sources.md` when new major sources inform repo standards.

## Skill Iteration

If a documentation research task exposes missing structure, add that guidance back into this skill or another repo-local skill. Treat the skill as a maintained workflow, not a one-time prompt.
