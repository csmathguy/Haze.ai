# UI Principles

## Tax Workflow Priorities

- Make review state obvious.
- Keep exact values visible without hover-only interactions.
- Show provenance, confidence, and validation near the field being reviewed.
- Favor tables, summaries, and side panels that support document comparison.

## MUI Usage

- Prefer theming and variants over one-off styling.
- Use Data Grid or dense table layouts when the user needs to reconcile many rows.
- Keep icon use functional, not decorative.

## Charts

- Use charts for trend, allocation, and discrepancy summaries.
- Keep a text or tabular fallback next to the chart.
- Avoid chart types that make exact comparison hard when the numbers drive tax decisions.

## Accessibility

- Preserve keyboard access for import, review, and export flows.
- Use clear labels and helper text for tax fields.
- Surface errors inline and in a summary area when forms are long.
